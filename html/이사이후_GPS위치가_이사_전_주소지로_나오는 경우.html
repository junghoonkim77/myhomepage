GPS 같은 경우 위성 특성상 실내에서는 정확도가 떨어지기 때문에 해당 위치를 보정하는데, 
Wi-Fi, 모바일 네트워크, 센서 정보를 활용하는 것이고
모바일 네트워크 정보를 활용한다고는 하지만(cellTower) 정보는 정확한 
위치조회에 별 도움이 되지 않기 때문에,
우리나라 같이 전국 방방 곡곡 Wi-Fi가 깔린 곳에서는 거의 
대부분 실내 위치는 WPS+GPS라고 보면 됩니다.
이사를 했는데 예전주소가 나온다 이건 무조건 WPS의 영향입니다. 
WPS에서 가져오는 위치정보는 두 가지입니다.

1. 구글 시설정보 시스템에서 해당 와이파이가 시설된 위치정보를 
가져 오는 것(공공 와이파이 등 시설 장소가 특정되는 것들)

2. 사용자 데이터베이스를 기반으로 WiFi AP 정보를 
수집한 것(보통 대부분의 홈와이파이/개인 와이파이)


1. 그냥 기다리는 것
그냥 기다리다 보면 고객이 평소에 GPS 켜고 다닌다면, 
예를들어 분명히 탄방동 649번지 주변에서 돌아다니고 있었는데, 
집으로 들어오니까 와이파이 잡히고 전혀 이상한 주소로 잡힌다.
그럼 구글 WPS에서 이상 정보를 감지하고, 
동일 사례가 반복되면 데이터베이스에 해당 AP 위치정보가 수정됨.
이걸 조금 더 빠르게 하려면 그냥 본인이 학습시켜주면 됩니다. 
창가같이 WiFi-GPS 둘 다 정보가 받아지는 곳에서 위치조회 
자주 하다보면 학습이 좀 더 빨라질 수는 있음.

2. Wi-Fi SSID랑 MAC을 둘 다 바꿔주는 것
타 사이트는 다를 수도 있으나, 구글 WPS API 사용하는 앱에서는 
구글은 이론 상 AP의 MAC을 식별기준으로 삼고 해당 AP의 
신호세기 등을 기준으로 위치를 판단하기 때문에
식별자인 SSID와 MAC을 다 바꾸면 해당 AP의 위치정보를 
활용할 수 없게되어, 위치가 정확하진 않을 수 있어도 
예전 집으로 나오진 않을 것임(두 가지 외 다른 식별자도 
추가 이용한다면 바로 적용안될 수 있음)
(유선에서 포트리셋하고 하는 거 그런 건 아무 상관없는데, 
그런 오안내 겸 삽질을 하고 있으면서 무선 탓 하는중)
여튼 왕도는 그냥 기다리는 거고, 따지고 들자면 
구글/단말기/AP제공업체(KT 홈 와이파이라면 KT 유선) 세 가지가 연관될 뿐이지
무선하고는 별 관계가 없는 이야기입니다.
