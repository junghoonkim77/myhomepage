RSRP (reference signals received power)
정해진 주파수 대역폭 내에 있는 Reference signal의 세기
3G / LTE 천이를 위해 사용됨

>=80 아주 좋음
-80~-90 좋음
-90~-100 슬슬 끊어짐
-100~ 통탈


 RSRQ (reference signal received quality)
 할당된 Resource Block에 대한 RSRP 와 RSSI의 비율
 RSSI와 RSRP가 강한 지역이라도 기지국간 

>=-10db  아주 좋음
-10db ~-15db 좋음
-15~-20 안좋음
-20~ 매우 안좋음 (나가리)


RSSI ( received signal strength indicator)
 LTE기준  / 3Gsms 아래 신호에 5~8정도더한 폭으로 보면 됨
 단말로 수신되는 전체 전파의 세기 / 간섭 ,노이즈를 포함한 신호세기

~-65dbm (아주 좋음)
-65 ~ -75 (좋음) 실제는 -90정도도 그럭 저럭 쓸만함
-75~-85 ( 안좋음) 
-85~95 ( 매우 안좋음)


SINR ( signal to interference plus noise ratio/ 신호대 잡음비) / SINR은 양수값임
단말이 받아야 되는 기지국 신호의 세기와 간섭(타 기지국 신호등)간의 비율

>=20db ( 매우 좋음)
13db~20db ( 좋음)
0db~13db ( 안좋음)
<=0 ( 나가리)


EC/IO (전체적인 잡음지수 , 주변 불요파등 포함)

0~ -6 ( 매우 좋음)
-7~ -10 ( 좋음 )
-11~-20 ( 안좋음)
