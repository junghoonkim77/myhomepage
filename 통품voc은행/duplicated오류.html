증상: 유심 기변 후 음성수발신불가

  확인사항:
  
  - KOS 상 문의 단말 IMEI값으로 유심이동성공한 상황
  
  - EIR DB 조회 시 해당 IMEI값 하기 블랙 상태
     
    STATUS = DUPLICATED(B)
  
  - 팀장석에서 비정상 기기변경 제한 대상 단말 대상로는 조회 안됬던 상황 / 
    용인지능망쪽에서도 블랙될 만한 내용없으나 블랙 상태임에 ITSM 접수 통해 
    단말 EIR 정보 WHITE로 전문 처리 1차 답변받음
  
  - ITSM에선 비정상 기기변경 차단
  (동일단말 3개이상의 계약으로 기변이력이 존재해여 차단)
   으로 ITSM측 조치불가 답변
  
  - 용인지능망측 재컨택 후 용인지능망팀에서 블랙리스트에서 해당 단말 블랙 정보 삭제 
  처리완료 답변받음
  
  - 이후에도 EIR 상 블랙 상태는 동일
  
  -상기 과정이후 팀장석에서 재조회 시 비정상 기기변경 제한 대상 단말로 조회되어 
  탐장석에서 해제 후 EIR NORMAL(W) 상태 확인 및 정상이용확인 
  
  -------------------------------------
  
  > 위 유사 케이스 인입 당시엔 팀장석에서 비정상 기기변경 차단 대상 단말로 
   조회되지않더라도 최종적으로 용인지능망 통해 블랙리스트에서 블랙정보 삭제 
   조치받았으나 EIR 상태 동일 경우
  
  용인지능망팀쪽 요청 후 다시 팀장석에서도
  비정상 기기변경 차단 대상로 조회되는지 재확인요청 필요 할 수 있음.
  
  * 또한 용인지능망측 처리 후 동일증상으로 용인 지능망 재문의 당시
  블랙단말리스트에서는 해당 단말 블랙 정보 삭제된 상태는 맞으나,
  기존에 실패했던 고객번호 유심 그대로 재시도 시 기존 맵핑된 정보에 따라 지속 실패될
  수 있어서 이 경우 문의단말에 잠시 타유심으로 이동 테스트 후 다시 문의 번호 
  고객유심으로 원복하여 확인 테스트 필요 답변받음
  
  이에 용인지능망 및 팀장석에서 블랙정보삭제 및 비정상 기기변경 차단해제 후 
  리셋시에도 EIR 상태 동일할 경우
  타유심으로 테스트 후 원복 테스트도 진행해볼 것.
